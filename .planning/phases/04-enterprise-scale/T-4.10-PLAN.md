## Step 2: Create SLMGuard (ONNX Model Wrapper)

**Files created:** `src/orchestra/security/guard.py`

**What to implement:**

`SLMGuard` is a standalone class that loads an ONNX-quantized prompt injection classifier and exposes a synchronous `classify()` method. It is NOT a guardrail itself -- it is the inference engine that `OutputScanner` (Step 3) delegates to.

### Class: `SLMGuard`

**Constructor `__init__(self, model_id: str | None = None, threshold: float = 0.5, device: str = "cpu")`**

- `model_id` defaults to `"gravitee-io/Llama-Prompt-Guard-2-86M-onnx"` (282 MB quantized ONNX).
- `threshold` is the probability above which text is classified as injection (default 0.5).
- `device` passed to ORT session options (always `"cpu"` for now -- no GPU in Orchestra's server runtime).
- Store these as instance attributes. Do NOT load the model in `__init__`.

**Lazy loading: `_ensure_loaded(self) -> None`**

- Guard with a `self._loaded: bool = False` flag.
- Import `from optimum.onnxruntime import ORTModelForSequenceClassification` and `from transformers import AutoTokenizer`.
- Load tokenizer: `AutoTokenizer.from_pretrained(self._model_id)`.
- Load model: `ORTModelForSequenceClassification.from_pretrained(self._model_id, file_name="model.quant.onnx")`.
  - For the ProtectAI fallback (`protectai/deberta-v3-base-prompt-injection-v2`), the file is `onnx/model.onnx` instead. Detect this by checking if `"protectai"` is in the model_id and use the appropriate `file_name` kwarg.
- Store as `self._tokenizer` and `self._model`.
- Set `self._loaded = True`.
- Wrap the entire load in a try/except. On `ImportError`, raise a clear error: `"PromptShield requires optimum[onnxruntime] and transformers>=4.51. Install with: pip install 'optimum[onnxruntime]' 'transformers>=4.51,<5.0'"`.
- On any HuggingFace download error, raise `RuntimeError` with the model_id and original traceback.
- Log model load via structlog: `guard.model_loaded`, model_id=, duration_ms=.

**Classification: `classify(self, text: str) -> SLMGuardResult`**

- Call `_ensure_loaded()`.
- Tokenize with truncation at 512 tokens (PG2 context window): `self._tokenizer(text, return_tensors="pt", truncation=True, max_length=512)`.
- Run inference: `outputs = self._model(**inputs)`.
- Extract logits, apply softmax: `import torch; probs = torch.softmax(outputs.logits, dim=-1)`.
  - NOTE: Since we are using ONNX Runtime (not PyTorch model), the outputs.logits will be a numpy array, not a torch tensor. Use `import numpy as np; probs = np.exp(logits) / np.sum(np.exp(logits), axis=-1, keepdims=True)` for softmax instead of torch. Do NOT add torch as a dependency.
- For Llama PG2 models, label index 2 = "INJECTION", index 1 = "JAILBREAK", index 0 = "BENIGN". For ProtectAI, index 1 = "INJECTION", index 0 = "SAFE".
  - Determine label mapping from `self._model.config.id2label`. Read the config's `id2label` dict. Find the index whose label contains "INJECTION" (case-insensitive). Sum that probability with any "JAILBREAK" label probability (for PG2 models that distinguish them).
- Return `SLMGuardResult(is_injection=injection_prob > self._threshold, score=float(injection_prob), label=predicted_label, model_id=self._model_id)`.

### Dataclass: `SLMGuardResult`

```python
@dataclass(frozen=True)
class SLMGuardResult:
    is_injection: bool
    score: float        # 0.0-1.0 probability
    label: str          # "BENIGN", "INJECTION", "JAILBREAK"
    model_id: str
```

### Constants

```python
DEFAULT_MODEL = "gravitee-io/Llama-Prompt-Guard-2-86M-onnx"
LIGHTWEIGHT_MODEL = "gravitee-io/Llama-Prompt-Guard-2-22M-onnx"
PROTECTAI_MODEL = "protectai/deberta-v3-base-prompt-injection-v2"
```

### Environment variable: `ORCHESTRA_GUARD_MODEL`

If `model_id` is `None` in the constructor, check `os.environ.get("ORCHESTRA_GUARD_MODEL")` before falling back to `DEFAULT_MODEL`. This lets operators switch models at deploy time without code changes.

### Thread safety note

`_ensure_loaded()` should use a `threading.Lock` to prevent concurrent double-loads when multiple async tasks call `classify()` simultaneously via `asyncio.to_thread()`. The lock should be a class-level or instance-level `threading.Lock()` acquired only during the load path (not during inference -- inference is read-only on the loaded model and tokenizer).

**Dependencies:** None (standalone module).

## Step 3: Create OutputScanner (Guardrail Protocol Implementation)

**Files created:** `src/orchestra/security/output_scanner.py`

**What to implement:**

`OutputScanner` implements the `Guardrail` protocol from `guardrails.py`. It wraps `SLMGuard` (Step 2) and optionally triggers the `Attenuator` (Step 4).

### Class: `OutputScanner`

**Constructor:**

```python
def __init__(
    self,
    guard: SLMGuard | None = None,
    model_id: str | None = None,
    threshold: float = 0.5,
    on_fail: OnFail = OnFail.BLOCK,
    attenuator: Attenuator | None = None,
) -> None:
```

- If `guard` is provided, use it directly (for testing / sharing a single loaded model across scanners).
- If `guard` is `None`, create one: `SLMGuard(model_id=model_id, threshold=threshold)`.
- Store `attenuator` for capability attenuation on detection (optional -- can be `None`).
- Store `on_fail`.

**Properties (required by Guardrail protocol):**

```python
@property
def name(self) -> str:
    return "output_scanner"

@property
def on_fail(self) -> OnFail:
    return self._on_fail
```

**`async def validate(self, text: str, **kwargs) -> GuardrailResult`:**

This is the core protocol method. It must:

1. Run classification in a thread pool (ONNX inference is synchronous, must not block the event loop):
   ```python
   result = await asyncio.to_thread(self._guard.classify, text)
   ```

2. Log the result via structlog:
   ```python
   logger.info(
       "output_scanner.classify",
       is_injection=result.is_injection,
       score=round(result.score, 4),
       label=result.label,
       model_id=result.model_id,
       text_length=len(text),
   )
   ```

3. If `result.is_injection is False`: return `GuardrailResult(passed=True, output=text)`.

4. If `result.is_injection is True`:
   - If `self._attenuator is not None` and `"context"` is in `kwargs`:
     - Call `self._attenuator.attenuate(kwargs["context"])` to restrict the agent's capabilities.
   - Create a violation:
     ```python
     violation = GuardrailViolation(
         guardrail=self.name,
         message=f"Prompt injection detected (score={result.score:.3f}, label={result.label})",
     )
     ```
   - Return `GuardrailResult(passed=False, output=text, violation=violation.message, violations=[violation])`.

**IMPORTANT:** The `context` kwarg is passed by `GuardedAgent` when it calls `self.output_guardrails.run(result.output)`. Currently `GuardedAgent` does NOT pass `context` as a kwarg to `chain.run()`. This requires a small modification to `GuardedAgent.run()` in Step 5 -- the scanner should gracefully handle its absence (check `"context" in kwargs`).

**Legacy interface methods (required by Guardrail protocol):**

```python
async def validate_input(self, *, messages, model=None, tools=None):
    return []  # OutputScanner only scans outputs

async def validate_output(self, *, output_text, model=None):
    result = await self.validate(output_text)
    return result.violations
```

**Dependencies:** Step 2 (SLMGuard), Step 4 (Attenuator -- but Attenuator is optional/None-safe, so the code can be written before Step 4 is complete; just import the type for the type hint).

**Tests to write (in `tests/unit/test_prompt_shield.py`, same file as Step 2):**

1. **`test_output_scanner_passes_benign`** -- Create OutputScanner with a mock SLMGuard that returns `is_injection=False`. Call `await scanner.validate("safe text")`. Assert `result.passed is True`.

2. **`test_output_scanner_blocks_injection`** -- Mock SLMGuard returns `is_injection=True, score=0.95, label="INJECTION"`. Assert `result.passed is False`, violation message contains "injection detected", score is present.

3. **`test_output_scanner_implements_guardrail_protocol`** -- `isinstance(OutputScanner(...), Guardrail)` returns True (runtime_checkable protocol).

4. **`test_output_scanner_name_and_on_fail`** -- `.name == "output_scanner"`, `.on_fail == OnFail.BLOCK` by default.

5. **`test_output_scanner_custom_on_fail`** -- `OutputScanner(on_fail=OnFail.LOG)` has `.on_fail == OnFail.LOG`.

6. **`test_output_scanner_calls_attenuator_on_injection`** -- Mock SLMGuard returns injection. Provide a mock Attenuator. Provide `context=ExecutionContext()` in kwargs. Assert `attenuator.attenuate` was called with the context.

9. **`test_output_scanner_in_guardrail_chain`** -- Create a `GuardrailChain([OutputScanner(guard=mock_guard)])`. Run it. Verify it integrates correctly (chain calls `validate`, respects `on_fail`).

## Step 4: Create Attenuator (Capability Attenuation)

**Files created:** `src/orchestra/security/attenuator.py`

**What to implement:**

`Attenuator` creates a restricted `ToolACL` and marks the `ExecutionContext` as restricted. It does NOT mutate the existing ACL (frozen dataclass). It creates a NEW one.

### Class: `Attenuator`

**Constructor:**

```python
def __init__(
    self,
    restricted_tools: set[str] | None = None,
    restricted_patterns: list[str] | None = None,
    allow_tools_in_restricted: set[str] | None = None,
) -> None:
```

- `restricted_tools`: Specific tool names to deny when attenuated. Default: `{"execute_code", "shell", "file_write", "http_request", "database_write"}` (a sensible default set of dangerous tools).
- `restricted_patterns`: Glob patterns to deny. Default: `["*_write", "*_delete", "*_execute", "shell_*", "code_*"]`.
- `allow_tools_in_restricted`: Explicit allowlist that survives attenuation (e.g., `{"read_file", "search"}` -- read-only tools the agent can still use). Default: `None` (only tools not matching deny rules are allowed).

**`def attenuate(self, context: ExecutionContext, current_acl: ToolACL | None = None) -> ToolACL`:**

1. Set `context.restricted_mode = True`.
2. Log the attenuation event:
   ```python
   logger.warning(
       "attenuator.capabilities_restricted",
       run_id=context.run_id,
       node_id=context.node_id,
       denied_tools=sorted(self._restricted_tools),
       denied_patterns=self._restricted_patterns,
   )
   ```
3. Create and return a NEW `ToolACL` instance:
   ```python
   return ToolACL(
       allowed_tools=self._allow_tools or set(),
       denied_tools=self._restricted_tools,
       allow_patterns=[],
       deny_patterns=self._restricted_patterns,
       allow_all=False if self._allow_tools else False,
   )
   ```
   - If `allow_tools_in_restricted` was provided, the new ACL has `allowed_tools=self._allow_tools, allow_all=False` -- only explicitly allowed tools pass.
   - If `allow_tools_in_restricted` is `None`, the ACL has `allow_all=False, denied_tools=..., deny_patterns=...` -- effectively denying the dangerous tools while allowing others that don't match the deny patterns. Actually this won't work because `allow_all=False` with empty `allowed_tools` denies everything. Correct approach: `allow_all=True, denied_tools=..., deny_patterns=...` -- allow everything EXCEPT the denied tools/patterns.

   **Corrected logic:**
   - If `allow_tools_in_restricted` is provided: strict allowlist mode.
     ```python
     ToolACL(allowed_tools=self._allow_tools, allow_all=False)
     ```
   - If `allow_tools_in_restricted` is `None`: denylist mode (allow everything except dangerous tools).
     ```python
     ToolACL(denied_tools=self._restricted_tools, deny_patterns=self._restricted_patterns, allow_all=True)
     ```

4. The caller (OutputScanner or GuardedAgent) is responsible for applying this new ACL to the agent. The Attenuator just creates it and marks the context. Return the new ACL so the caller can use it.

**`def is_attenuated(self, context: ExecutionContext) -> bool`:**

Simple check: `return context.restricted_mode`.

### Class method: `Attenuator.default()`

Factory that creates an Attenuator with the sensible defaults described above.

**Dependencies:** Step 1 (restricted_mode on ExecutionContext).

**Tests to write (in `tests/unit/test_prompt_shield.py`, same file):**
